DNScode is a project to help simplify DNS zone management, when using plain text files with servers like BIND and NSD. It provides a framework for programmatically generating zone files with Python, allowing for more flexability, compared to other DNS as code solutions.\hypertarget{index_autotoc_md4}{}\doxysubsection{\texorpdfstring{Installation}{Installation}}\label{index_autotoc_md4}

\hypertarget{index_autotoc_md5}{}\doxysubsection{\texorpdfstring{Usage}{Usage}}\label{index_autotoc_md5}
Import the dnscode package into a python script, create a zone object, then add records into the zone. Records can be added either via helper functions in the \href{https://dnscode.minecraftchest1.us/classdnscode_1_1dnscode_1_1Zone}{\texttt{ dnscode.\+Zone}} class, or by manually creating record objects and adding them through \href{https://dnscode.minecraftchest1.us/classdnscode_1_1dnscode_1_1Zone\#a338bc686b7c7db2cab7827996a3f23f3}{\texttt{ dnscode.\+Zone.\+add()}}. Both methods are shown in the example below.

Once the zone is setup, you can save it as a text file using \href{https://dnscode.minecraftchest1.us/classdnscode_1_1dnscode_1_1Zone\#adfe5442ed2137a324f1c5ba676ba2043}{\texttt{ dnscode.\+Zone.\+save\+\_\+file()}}. Currently, it also outputs to to STDOUT. See \href{https://code.minecraftchest1.us/minecraftchest1/dnscode/issues/5}{\texttt{ https\+://code.\+minecraftchest1.\+us/minecraftchest1/dnscode/issues/5}} for details.

API docs at \href{https://dnscode.minecraftchest1.us/classdnscode_1_1dnscode_1_1Zone}{\texttt{ https\+://dnscode.\+minecraftchest1.\+us/classdnscode\+\_\+1\+\_\+1dnscode\+\_\+1\+\_\+1\+Zone}}
